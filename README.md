## Basic Arithmetic Expression Compiler is a C++ application that simulates the core phases of a compiler by parsing and evaluating simple arithmetic expressions.
This project focuses on fundamental compiler design concepts such as lexical analysis, syntax parsing, and expression evaluation.
The program takes arithmetic expressions (e.g., 3 + 5 * (2 - 1)) as input, processes them through different compilation stages, and outputs the computed result. It demonstrates how high-level expressions are analyzed and executed by a compiler internally. The implementation typically uses techniques like tokenization and parsing (e.g., recursive descent or stack-based parsing) to evaluate expressions correctly while maintaining operator precedence and associativity.

## Objectives
To understand the basic phases of a compiler
To implement lexical analysis (tokenization)
To parse arithmetic expressions using appropriate parsing techniques
To evaluate expressions based on precedence rules
To simulate compilation and execution in a simplified manner

## Tools and Technologies Used
Programming Language: C++
Compiler: MinGW
IDE/Editor: Visual Studio Code

## Features
Accepts arithmetic expressions as input
Supports operators: +, -, *, /
Handles parentheses for precedence control
Implements tokenization (numbers, operators, symbols)
Parses expressions using recursive descent or stack-based parsing
Evaluates expressions and outputs results
Detects basic syntax errors (invalid expressions, division by zero)
